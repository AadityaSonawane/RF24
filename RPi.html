Linux General/Raspberry Pi

RF24 supports a variety of Linux based devices via various drivers. Some boards like RPi can utilize multiple methods to drive the GPIO and SPI functionality.

Potential PreConfiguration

If SPI is not already enabled, load it on boot:

sudo raspi-config

A. Update the tool via the menu as required
B. Select Advanced and enable the SPI kernel module
C. Update other software and libraries

sudo apt-get update
sudo apt-get upgrade


Build Options

The default build on Raspberry Pi utilizes the included BCM2835 driver from http://www.airspayce.com/mikem/bcm2835

  1. See the Linux section for automated installation
  2. Manual install:
    make; sudo make install


Connections and Pin Configuration

Using pin 15/GPIO 22 for CE, pin 24/GPIO8 (CE0) for CSN

Can use any available SPI BUS for CSN.
In general, use

RF24 radio(<ce_pin>, <a>*10+<b>);

for proper constructor to address correct spi device at /dev/spidev<a>.<b>
Choose any GPIO output pin for radio CE pin.

General:

RF24 radio(22,0);

MRAA Constructor:

RF24 radio(15,0);

See http://iotdk.intel.com/docs/master/mraa/rasppi.html

SPI_DEV Constructor

RF24 radio(22,0);

https://www.raspberrypi.org/documentation/usage/gpio/

Pins:

PIN NRF24L01 RPI RPi -P1 Connector
1 GND rpi-gnd (25)
2 VCC rpi-3v3 (17)
3 CE rpi-gpio22 (15)
4 CSN rpi-gpio8 (24)
5 SCK rpi-sckl (23)
6 MOSI rpi-mosi (19)
7 MISO rpi-miso (21)
